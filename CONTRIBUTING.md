# Contributing

This repository follows a research-oriented engineering workflow focused on scalable AI systems, cloud-native infrastructure, and operational ML deployment patterns.

## Development Standards

- Keep credentials and secrets out of version control
- Document infrastructure changes clearly
- Validate Python syntax before pushing changes
- Track benchmark-related modifications when changing inference logic
- Add experiment notes when evaluating new workflows or deployment configurations
- Maintain reproducibility for benchmarking and evaluation workflows

## Commit Practices

Use descriptive commit messages that clearly explain infrastructure, inference, orchestration, benchmarking, or deployment-related changes.

## Infrastructure Notes

The repository includes AWS-oriented workflows, Docker-based execution support, Kubernetes deployment manifests, CI validation workflows, and event-driven processing utilities. Changes affecting infrastructure or deployment logic should include corresponding documentation updates.
